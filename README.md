# cmdtyper

> 🐧 Linux 命令行交互式教学系统 — 终端模拟打字 · 体系化学习 · 模拟输出反馈

**cmdtyper** 是一个基于 Rust 的终端 TUI 应用，面向 Linux 初学者到中级用户，提供沉浸式的命令行教学体验。它模拟真实终端环境进行打字练习，包含体系化的命令、符号、系统架构教学课程，并通过模拟输出让你直观看到命令的执行效果——完全不执行任何真实命令。

## ✨ 特性

- **🖥️ 终端模拟打字** — 像真正的终端一样练习，prompt + 命令 + Enter 换行
- **📖 体系化学习中心** — 命令专题 / 符号专题 / 系统架构，二级菜单导航
- **📺 模拟命令输出** — 打完命令后展示逼真的模拟终端输出
- **📝 默写模式** — 看中文描述写命令，多答案智能匹配
- **📊 统计面板** — WPM、准确率、字符分析、类别掌握度
- **🔒 绝对安全** — 不执行任何真实命令，所有输出均为预置数据

## 🎮 模式

### ⌨️ 对着打 — 终端模拟

模拟真实终端环境，打完一条命令按 Enter 进入下一行：

```
user@cmdtyper:~$ ls -la /var/log          ← 已完成（绿色）
user@cmdtyper:~$ grep -r "error" /var/█   ← 正在输入（光标）
─────────────────────────────────────────
  搜索日志中的 error     WPM: 42  准确: 96%
```

- 已完成命令留在屏幕上（像真正的终端历史）
- 底栏含义提示可按 `H` 切换
- 支持多行命令显示

### 📖 学习中心

二级菜单，体系化学习：

```
📖 学习中心
├── 命令专题 — 按类别系统学习 (文件操作/文本处理/网络/Git/Docker...)
├── 符号专题 — |, >, *, $, &&, 正则表达式...
├── 系统架构 — 目录结构/权限/systemd/网络/包管理/配置文件
└── 专题复习 — 知识梳理 + 集中练习
```

每个命令学习分三阶段：
1. **概览** — 语法、选项、详细讲解
2. **演示** — 实例命令 + 模拟终端输出
3. **跟打** — 逐字符练习，完成后展示模拟输出

### 📝 默写模式

只给中文描述，你凭记忆写命令：
- 多答案匹配（`ls -la` 和 `ls -al` 都算对）
- 智能标准化（忽略多余空格）
- 错误时展示最接近答案 + diff 高亮

### 📊 统计面板

- 速度总览（平均/最佳 WPM）
- 字符分析（每个字符的速度和错误率）
- 类别掌握度（各专题进度条）
- 练习日历（连续天数统计）

## 🚀 快速开始

### 从源码构建

```bash
git clone https://github.com/ZhaoLingSun/cmdtyper.git
cd cmdtyper
git checkout v0.2
cargo build --release
./target/release/cmdtyper
```

### Docker 运行

```bash
docker compose build
docker compose run --rm cmdtyper
```

### 依赖

- Rust 1.82+ (edition 2024)
- 支持 256 色的终端

## ⌨️ 快捷键

| 按键 | 操作 |
|------|------|
| `↑` `↓` | 导航菜单 |
| `←` `→` | 切换难度 / 翻页示例 |
| `Enter` | 确认 / 提交 / 下一题 |
| `H` | 切换含义提示（对着打模式） |
| `Tab` | 跳过当前命令 |
| `Ctrl+R` | 重练当前命令 |
| `Esc` | 返回上一级 |
| `q` | 退出 |

## 🔒 安全

**cmdtyper 不执行任何真实命令。** 所有命令输出均为 TOML 文件中的预置模拟数据。没有 shell 调用、没有 `Command::new()`、没有 `system()`。
